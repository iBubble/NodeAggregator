# 更新日志 (CHANGELOG)

所有对项目的显著修改都将记录在此文件中。

## [未发布] - 2026-02-08

### 新增
- **Linux.do 节点获取统计**: 定时任务日志现在会记录各来源（Github、网站、Linux.do）获取的节点数量。
  - 新增 `sources` 字段记录各来源抓取节点数
  - 新增 `linuxdoValid` 字段记录 Linux.do 节点验证通过数量
  - 验证完成后输出 Linux.do 节点验证结果统计日志
- **界面精简**: 移除了 "仅更新 Github 节点" 按钮及其相关逻辑，完全整合至 "全网获取" 功能中。
- **功能调整**: 将 "服务器直连测试" 按钮更新为 "节点本地检测"，并链接至新的独立测试页面 (NodeLocalChecker)。

## [未发布] - 2026-02-07

### 优化
- **服务器运行时间显示位置调整**: 将服务器运行时间从独立的状态栏移动到 Header 右侧，放置在 Aggregator.yaml 下载按钮前方，形成更紧凑的布局。
- **状态同步逻辑修复**: 修复了页面加载时服务器运行时间无法显示的问题，现在无论服务器状态如何都会正确获取并显示运行时间信息。

## [未发布] - 2026-02-06

### 新增
- **定时任务日志系统**: 实现了定时任务执行详情的记录功能。
  - 后端：增加 `/api/cron_logs` 接口，将任务耗时、状态、节点数量等存入 `cron_logs.json`。
  - 前端：增加定时任务日志查看弹窗，支持显示下次执行时间及格式化耗时展示。

### 优化
- **项目结构清理**: 移除了冗余的备份文件（`.tar.gz`）、过时的测试脚本及临时数据。
- **UI/UX 改进**:
  - 整合 "Linux.do 导入" 功能至 "全网获取" (Free Style) 流程中。
  - 优化首页按钮布局，提升操作流程度。
  - 更新项目 README，增加详细的技术栈说明、目录结构及 API 文档。

### 修复
- **节点检测逻辑**: 优化了连通性测试与纯净度检测的判定算法，解决了部分节点无法正确显示或检测超时的问题。
- **清除功能**: 修复了"清空所有节点"确认框自动关闭的 Bug。
